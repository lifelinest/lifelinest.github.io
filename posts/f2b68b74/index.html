• 阐述清楚问题就解决了一半
  • 一致的体验是更好的体验
  • 创造允许犯错，设计要知道保留哪些
  • 想让用户爱上你的设计，先爱上你的用户
  • 好的设计是我们与竞争对手保持差距最重要的方式
  • 如果你觉得好的设计是昂贵的，你应该看看坏设计的成本
  • 像其他设计一样，视觉设计也是解决问题，不是个人喜好
  • 因为每个人都知道自己喜欢什么，所以每个人都觉得自己是用户界面专家
  • 易用性也许不可见，但却不可缺席
  • 如果我有1000个想法，即使最终只有一个实现了，我也会非常满意
  • 如果你发现你界面中某个元素需要说明，你需要重新设计
  • 简化意味着去掉不必要的元素，让必要元素凸显
  • 天才就是将复杂问题简单化的人
  • 不用盯着空白太久，勇敢的跨出第一步，其他的自然而然就会发生

    如何搭建 Cursor Skill 标准化目录结构与部署环境

    如何搭建 Cursor Skill 标准化目录结构与部署环境

    Lifeline2026-06-192026-07-11

    前言

    上一讲我们系统吃透了Skill、Rule、Token三大核心基础概念，完成了零基础理论认知铺垫。从本篇开始，我们正式进入Skill实操落地阶段，从零搭建可直接使用的AI技能运行环境。

    绝大多数新手遇到的Skill失效、无法触发、执行错乱等问题，90%都源于路径错误、目录不标准、命名不规范。AI编程助手的Skill是一套标准化结构化能力包，对存放路径、目录结构、文件格式有严格的官方约束，只要不满足规范，AI就无法扫描、识别和加载技能。

    本篇将全程手把手实操教学，带你完整搭建Skill运行环境，深度拆解两种部署模式、官方标准目录结构、核心文件硬性规范，同时汇总新手高频踩坑点，夯实后续所有Skill创作、调试、落地的核心基础。

    一、Skill 部署核心逻辑：生效范围决定部署方式

    目前主流AI编程助手（Claude Code、CodeBuddy、Cursor）统一提供两套Skill部署方案，分为用户级（全局生效）和项目级（局部生效）。两种模式的核心差异集中在生效范围、存储路径、适用场景，大家可根据自身使用需求灵活选用。

    核心**取舍口诀：**通用技能放全局，项目专属放本地。

    1.1 用户级 Skill（全局生效）

    用户级Skill绑定当前电脑系统用户，存放于系统用户根目录下，本机所有项目均可自动加载、全局生效，无需重复新建、配置技能。

    适配场景：通用高频复用技能，例如代码格式化、单元测试自动生成、文本内容润色、项目临时日志清理等通用自动化操作。

    三大主流工具官方全局路径（Windows/Mac/Linux全平台通用）：

    • CodeBuddy：~/.codebuddy/skills/
    • Claude Code：~/.claude/skills/
    • Cursor：~/.cursor/skills/ 或 ~/.agents/skills/

    路径符号说明：统一路径标识 ~ 代表系统当前用户根目录，无需手动逐级查找文件夹，终端和AI工具可自动识别匹配路径。

    1.2 项目级 Skill（局部生效）

    项目级Skill存放于当前项目根目录，仅对当前项目生效，切换至其他项目后自动失效。核心优势是可跟随项目文件****托管至Git，支持团队同步、共享、统一使用。

    适配场景：项目专属定制化技能，例如当前项目代码版本迁移、项目专属编码规范、定制化配置模板生成、项目专属脚本自动化等。

    三大主流工具官方项目路径：

    • CodeBuddy：项目根目录 /.codebuddy/skills/
    • Claude Code：项目根目录 /.claude/skills/
    • Cursor：项目根目录 /.cursor/skills/ 或 /.agents/skills/

    1.3 两种部署方式对比总结

    部署类型 生效范围 适用场景 是否可团队共享 推荐新手使用
    用户级（全局） 本机所有项目 通用高频技能、测试新Skill 否（仅本机生效） ✅ 优先推荐
    项目级（局部） 仅当前项目 项目专属规范、定制化迁移脚本 ✅ 可随Git同步团队 后期进阶使用

    二、Skill 标准目录结构（官方规范）

    Skill有一条强制官方规范：一个技能对应一个独立专属文件夹，严禁多个不同Skill共用同一个文件夹，避免AI加载错乱、规则冲突。官方提供两套标准目录结构，分别适配新手入门简单场景、企业级复杂商用场景。

    2.1 最简结构（新手首选）

    该结构为新手首选，适配单流程、轻量化、简单自动化技能，无需额外脚本、参考文档，仅依靠核心配置即可正常运行，覆盖90%的新手入门练习场景。

    结构说明：技能所有执行规则、操作指令、案例示例、执行流程，全部统一编写在SKILL.md 中，结构轻量化、零冗余、易维护，完全适配新手练手、简单功能开发。

    2.2 进阶复杂结构（企业级标准）

    该结构为企业级商用标准，适配多步骤长流程、批量自动化、需要脚本辅助、依赖规范模板的复杂技能。所有子文件夹遵循**「按需创建、可选补充」**原则，无需强行搭建空目录。

    2.3 各目录详细作用（小白必懂）

    • SKILL.md（核心必需）：Skill唯一核心载体，承载技能配置、执行规则、操作步骤、参考示例，无该文件则AI无法识别、加载技能
    • scripts（可选）：存放批量处理脚本、终端执行脚本（sh/py/bat），通过外置脚本简化正文内容，大幅降低Token占用，提升执行稳定性
    • references（可选）：存放长篇官方规范、业务文档、接口说明等参考资料，将大体积文本外置，避免正文臃肿超标
    • assets（可选）：存放项目初始化模板、固定配置文件、JSON模板等静态资源，供AI执行任务时按需调用

    三、核心文件 SKILL.md 硬性规范要求

    SKILL.md 是AI识别、加载、执行Skill的唯一核心文件，拥有严格的官方硬性规范，无任何容错空间，新手必须严格遵守，避免技能直接失效。

    3.1 基础命名规范

    • 文件名强制大写固定：必须为 SKILL.md，小写、大小写混搭（skill.md / Skill.md）均会导致加载失败
    • 文件后缀强制规范：仅支持 .md 格式，禁止使用txt、word、wps等其他格式文件

    3.2 文件核心特性

    • 唯一性约束：单个Skill文件夹内有且仅有一个 SKILL.md，禁止多文件混搭、重复创建
    • 加载优先级：AI加载Skill时会优先读取该文件，文件缺失直接判定技能失效
    • 跨工具通用性：规范适配所有主流AI编程助手，跨工具使用无需修改文件格式

    3.3 基础内容规范（前置要求）

    空白文件、乱码文件、纯注释无有效内容的 SKILL.md 都会导致技能加载异常、解析失败。合法可用的核心文件，必须包含两大基础模块：顶部YAML配置头 + 完整正文规则内容，后续章节将详细拆解全套编写方法。

    四、零基础编辑器选型与基础使用

    编写SKILL.md无需复杂专业工具，仅需要支持Markdown渲染、YAML语法高亮、纯文本编辑的软件。这里精选3款零基础友好工具，大家可按需选择使用。

    4.1 推荐编辑器（优先级从高到低）

    • VS Code（首选推荐）：免费开源、全平台适配，自带Markdown/YAML语法校验、实时预览、编码纠错功能，是编写、调试Skill的最优工具
    • Typora：纯可视化Markdown编辑器，零基础上手零门槛，专注文档排版梳理，适合新手整理Skill整体结构
    • 系统记事本：仅适合临时快速简单修改，无语法校验功能，容易出现格式错误，不推荐长期编写Skill使用

    4.2 编辑器基础使用禁忌

    • 严禁使用Word、WPS编辑：这类软件会自动生成隐藏格式、冗余符号，极易导致YAML配置解析失败、技能加载异常
    • 统一编码规范：所有文件必须使用 UTF-8** 编码**，杜绝中文乱码、格式解析异常问题

    五、新手致命避坑：命名、路径、格式禁忌

    本节汇总新手99%会踩的前期致命坑点，均为官方硬性禁忌规范，严格遵守即可规避绝大多数Skill失效、无法触发、解析报错等问题。

    5.1 文件夹命名禁忌（Skill文件夹名）

    ✅ 官方标准规范：文件夹名统一使用纯小写英文 + 连字符 -，命名简洁语义化，贴合技能核心功能

    ❌ 绝对禁止以下命名方式：

    • 禁止中文、中文符号、英文空格（错误示例：代码测试、test skill）
    • 禁止大写英文、下划线分隔（错误示例：Test-Go、go_test）
    • 禁止特殊符号、数字开头（错误示例：123test、go@test）

    合规****正确示例：go-test-gen、python-code-format

    5.2 路径禁忌

    • 禁止自定义随意路径：必须严格匹配对应AI工具的官方全局/项目路径，自定义路径无法被AI扫描识别
    • 禁止中文路径：所有上级目录、Skill文件夹全程无中文，避免编码兼容问题导致加载失败
    • 禁止目录层级过深：Skill文件夹直接放置在skills根目录下，不嵌套多余子目录，减少识别异常概率

    5.3 文件格式禁忌

    • 禁止修改核心文件名：仅支持固定名称 SKILL.md，任何大小写、后缀修改都会导致技能失效
    • 禁止YAML缩进使用Tab键：仅支持空格缩进，Tab缩进会直接导致配置解析报错
    • 禁止空文件、纯注释文件：无有效规则内容的文件，会导致技能加载异常、无法执行

    六、本篇总结 & 学习目标

    6.1 核心知识点总结

    1. Skill分为用户级全局生效、项目级局部生效两种部署模式，通用高频技能用全局模式，项目专属技能用局部模式；
    2. 官方提供最简单文件、进阶多目录两套结构，新手优先使用轻量化最简结构，复杂场景再升级企业级结构；
    3. SKILL.md 是唯一核心必需文件，其命名、格式、存放路径必须严格遵循官方规范；
    4. Skill创作前期规范大于内容，路径、命名、格式合规，是技能正常生效的核心前提。

    6.2 最终学习目标

    完成本篇学习后，你可以：

    • 精准区分两种部署模式，能够根据使用场景快速选择适配的部署路径；
    • 可独立创建标准化Skill文件夹与核心文件，完整搭建合规的Skill运行环境；
    • 熟练规避新手高频踩坑点，保障Skill基础环境100%合规可用。

    课后实操练习

    1. 对应自己使用的AI编程工具，找到官方全局Skill路径，新建一个命名完全合规的空白技能文件夹；

    2. 在新建文件夹内创建标准 SKILL.md 空白文件，逐一检查文件名、后缀是否完全合规；

    3. 手动搭建一套项目级Skill目录，对比全局、局部两种部署方式的差异，熟练掌握环境搭建规范。
